## 1. CMS（內容管理系統）需求

### 1.1 旅遊指南 CMS（A8.7 / FR-606 / FR-702）

- **功能編號**: FR-606（Should）、FR-702（Should）
- **管理對象**: 旅遊指南文章，由前端靜態頁面升級為後台可管理內容
- **後台能力**:
  - 發布 / 編輯 / 下架文章
  - 支援圖片上傳
  - 支援分類標籤（category / tags）
  - 支援 SEO meta 設定
- **前台呈現**:
  - 路徑 `/guide`
  - 依分類 / 標籤呈現文章列表
  - 文章內嵌「立即查看相關物業」CTA，串連內容與轉換漏斗
- **內容主題**: 香港人馬爾代夫攻略，包括簽證、轉機航班、保險、機場貴賓室、行李清單
- **資料表**: 新增 `cms_articles`（`id, slug, title_zh, content_zh, cover_image, category, status, published_at`）
- **API 端點**:
  - `/api/admin/cms/articles`（GET/POST/PUT/DELETE）：後台管理
  - `/api/cms/articles`（GET）：前台公開讀取

### 1.2 與 v2 的 CMS 差異

- v2 的 CMS 主要服務「旅遊指南 / 投資教育軌跡」，文章分類為「島嶼生活、投資入門、法規須知」，具投資導向。
- b6d4d4 版本移除投資教育內容，CMS 完全聚焦於港客旅遊攻略與在地化內容運營。

## 2. 後端管理界面需求

### 2.1 管理員後台功能模組（A8.6）

沿用四層角色模型（訪客 / 會員 / 管理員 / 超級管理員），管理後台集中在 `/member` 三合一頁面，按角色動態渲染。

| 編號 | 需求描述 | 優先級 |
|------|----------|--------|
| FR-601 | 訂單管理：列表 / 篩選 / 詳情、確認或拒絕、狀態流轉、人手覆核高單價訂單 | M |
| FR-602 | 旅客諮詢：即時收件匣、回覆、標記狀態（新 / 處理中 / 已結案）、轉化為跟進工單 | M |
| FR-603 | 房型庫存：即時調整可售房數、開關特定日期（維修 / 包場）、批量設定季節價格 | M |
| FR-604 | 帳戶管理（超級管理員限定）：新增 / 停用 / 刪除管理員、角色指派、操作日誌 | M |
| FR-605 | 優惠券 / 促銷管理：建立折扣碼、設定使用條件與有效期、查看使用成效 | S |
| FR-606 | 旅遊指南 CMS：發布 / 編輯 / 下架文章，支援圖片上傳與分類標籤 | S |
| FR-607 | 營運儀表板：即時訂單量、營收、入住率、熱門物業等視覺化報表 | S |

### 2.2 權限矩陣（A7）

- 訪客 / 會員 / 管理員 / 超級管理員四層角色。
- 較原始版本新增「禮賓加值服務」「優惠券管理」「通知中心」三項功能的權限歸屬。
- 超級管理員專屬：管理員帳戶增刪改、系統設定 / 支付網關金鑰管理。

### 2.3 技術實現（B8.2）

- 於 Hono 路由層建立統一的 RBAC 權限中介函式，依序驗證 Session / JWT、查詢 `admins` 表角色、依路由所需最低權限放行或回傳 403。
- 所有 `/api/admin/*` 路由強制套用管理員中介層，`/api/admin/accounts/*` 進一步疊加超級管理員專屬中介層。
- 新增 `audit_logs` 表記錄所有管理員寫入操作（`admin_id, action, target_table, target_id, before_json, after_json, created_at`），支援不可竄改（write-once）儲存策略。

### 2.4 與 v2 的後台差異

- v2 後台重點為「訂單、庫存、內容管理」，並強調「潛在客戶評分、CRM 階段標籤、IRM 分配、合規審批」。
- b6d4d4 版本保留訂單 / 庫存 / 內容管理，但移除 IRM（投資關係經理）工作流、投資合規審批、KYC/AML 相關流程，新增優惠券管理、營運儀表板、審計日誌。

## 3. 客戶管理系統需求

### 3.1 會員體系（A8.4 / FR-401–FR-405）

會員專區為三合一頁面 `/member`，按登入狀態與角色動態渲染：

- **未登入**: 會員權益介紹 + 登入 / 註冊（Email、Google、Apple 第三方登入）
- **一般會員**:
  - 個人資料管理
  - 消費統計
  - 會員等級進度條（新增）
- **我的預訂**: 訂單列表、狀態追蹤（待確認 / 已確認 / 已完成 / 已取消）、取消申請
- **電子憑證庫**: 可下載 PDF / 加入 Apple Wallet、行程出發前自動提醒
- **會員分級制度（新增）**: Coral（珊瑚）／ Pearl（珍珠）／ Atoll（環礁）三級，依年度消費自動升降級，享受房型優先鎖位、生日禮遇

### 3.2 會員等級計算（B5.4 / B4.2）

- 每日排程任務重新計算每位會員過去 12 個月消費總額。
- 依 `membership_tiers` 門檻更新 `user_membership.tier_id`。
- 等級升降變更時觸發通知服務發送對應禮遇通知。

### 3.3 諮詢與工單（FR-602 / B4.1）

- 旅客諮詢轉化為跟進工單。
- `inquiries` 表新增 `assigned_admin_id`、`priority`（VIP 旗標）欄位。
- VIP（環礁會員）訂單於後台自動標記「優先處理」旗標，可設定專屬客戶經理（account owner）。

### 3.4 與 v2 的客戶管理差異

- v2 的客戶管理圍繞「旅程階段標籤」（Leisure → Feeling → Desiring → Consulting → Owner → Advocate）與 IRM 分配，強調投資轉化漏斗。
- b6d4d4 版本的客戶管理回歸「會員分級 + 消費紀錄 + 諮詢工單」，以服務度假預訂與高端禮賓體驗為核心，無 IRM / 投資階段標籤。

## 4. WhatsApp 推廣功能需求

### 4.1 通知渠道（B3 / B5.1 / A10.5）

- 通知服務由 Edge API 層於訂單狀態變更、行前提醒等事件觸發時非同步呼叫。
- 通知渠道包括 **WhatsApp Business API** 與電郵服務（Resend / SendGrid）。

### 4.2 訂單與憑證通知（FR-304 / B5.1）

- 支付成功後自動產生訂單編號、電子憑證（含 QR Code 核銷碼）。
- 發送電郵 **與 WhatsApp** 通知給旅客。
- WhatsApp 沿用 iOS WhatsApp deep link 標準：
  - `wa.me` 連結
  - `encodeURIComponent` 訊息內容
  - `window.location.href` 同步導向
  - 未安裝 WhatsApp 時提供手動複製備援

### 4.3 行前關懷序列（A10.5 / B5.4）

- 到站前自動觸發的「行前關懷」郵件 / WhatsApp 序列：
  - **D-7**、**D-3**、**D-1**
  - 內容包含天氣、行李建議、機場貴賓室資訊
- 會員生日 / 結婚紀念日自動觸發專屬優惠或手寫風格電子卡片。

### 4.4 通知偏好設定（B8.1 / B6.2）

- 會員可於 `/member` 個人中心設定通知偏好。
- API 端點 `/api/notifications/preferences`（GET/PUT）支援會員隨時退訂行銷類通知。
- 行銷類通知採用明確的選擇加入（opt-in）機制，符合 PDPO 要求。

### 4.5 資料表

- 新增 `notifications_log`（`id, user_id, channel, template, status, sent_at`）記錄電郵 / WhatsApp 通知發送紀錄與狀態。

## 5. 「島主」/「海島業主」/「業主」定義與品牌故事

### 5.1 關鍵發現

在 `StayIslandsHK_PRD_SAD_b6d4d4.docx` 全文中，**並未出現「島主」或「海島業主」字樣**，也**未保留任何物業投資 / 業主權益相關業務流程**。

該文件將平台重新定義為單純的「香港高端馬爾代夫度假預訂平台」，品牌故事圍繞：

- **品牌主張**: 「由香港出發，以管家之心，款待每一段島嶼旅程」
- **兩大支柱**:
  1. 在地化信任（中文溝通、港幣定價、香港時區客服）
  2. 儀式化奢華（影片敘事、個人化行程、電子憑證如同精品開箱體驗）
- **角色模型**: 僅保留訪客、會員、管理員、超級管理員四層，無「業主」或「準業主」角色。

### 5.2 與 v2 的業主概念對比

v2.0 文件則大量圍繞「業主」與「物業投資」展開：

- **產品定位**: 「香港高端馬爾代夫度假與物業投資平台」
- **品牌口號**: 「Stay. Feel. Own. Earn — 先住進島嶼，再擁有島嶼」
- **用戶漏斗**: Stay → Feel → Desire → Own → Earn
- **業主相關功能**:
  - 業主專區（`/owner`）：收益、預訂、文件、IRM 對話
  - 投資試算機（`/invest/calculator`）
  - 業主收益預覽、資產願景清單
  - 業主租賃管理與收益分配流程
  - 業主後台查看入住日曆與租賃表現
- **用戶角色**: 訪客 / 會員 / 準業主 / 業主 / 管理員 / 超級管理員
- **資料表**: `owner_rental_records`、`ownership_shares`、`property_units` 等含 `owner_user_id`、`net_distribution`、`owner_usage_days` 等欄位
- **業主類型**: 被動收入導向業主（Income Owner）等

**結論**: b6d4d4 版本相當於把 v2.0 的「物業投資 / 業主」整條業務線移除，回歸純粹的高端度假預訂平台定位。

## 6. 與上一版 PRD（StayIslandsHK_PRD_SAD_v2.docx）的主要差異

### 6.1 核心定位轉變

| 維度 | v2.0 | b6d4d4 |
|------|------|--------|
| 平台定位 | 度假與物業投資平台 | 純度假預訂平台 |
| 品牌口號 | Stay. Feel. Own. Earn | Stay. Feel. Unwind |

| 收入來源 | 三層：住宿、物業銷售、租賃管理 | 住宿預訂、加值服務、優惠促銷 |

- **投資模組**: 投資專區（`/invest`）、投資試算機、投資意向表、投資教育內容
- **業主模組**: 業主專區（`/owner`）、業主後台、業主收益預覽、租賃記錄、收益分配
- **IRM 工作流**: 投資關係經理分配、諮詢轉工單給 IRM、CRM 投資階段標籤
- **合規審批**: KYC/AML、風險披露確認、投資合規審批流程
- **雙身份房間頁**: 同時服務度假者與投資者的產品頁面設計

### 6.3 新增或強化的功能模組

- **CMS 化旅遊指南**: 後台可發布 / 編輯 / 下架文章（v2 雖有 CMS，但內容主軸不同）
- **優惠券 / 促銷管理**: 新增折扣碼、使用條件、有效期、成效追蹤
- **會員分級制度**: Coral / Pearl / Atoll 三級會員，依年度消費自動升降級
- **營運儀表板**: 即時訂單量、營收、入住率、熱門物業等視覺化報表
- **審計日誌**: 管理員操作全量記錄（`audit_logs`）
- **行前關懷通知序列**: D-7 / D-3 / D-1 自動觸發郵件 / WhatsApp
- **庫存並發鎖定**: `locked_inventory` 欄位 + 15 分鐘支付逾時釋放機制
- **電子憑證開箱體驗**: 下單完成頁動畫化，含 QR Code 核銷碼
- **禮賓加值服務**: 機場接送、水上飛機、蜜月布置等可加購服務

### 6.4 技術架構差異

## 7. 其他關鍵需求摘要

### 7.1 首頁與品牌體驗（A8.1 / A10）

- 全螢幕循環影片 / 高畫質動態背景，行動數據 / 低頻寬時自動降級為靜態圖
- 即時顯示馬爾代夫當地時間與天氣，同步顯示香港時間雙時區對照
- 視差捲動（parallax scroll）品牌故事段落
- 深海軍藍（#1B2A4A）+ 香檳金（#B8902F）+ 珊瑚白 / 沙灘米色視覺識別系統
- 磁吸式 hover 效果、浮動標籤表單、品牌化骨架屏、水波紋 / 羅盤指針 loading 動畫

### 7.2 預訂與支付（A8.3 / B7 / B9）

- 下單流程：選房 → 日期 / 人數 → 加值服務 → 訂單摘要 → 港幣結算 → 支付 → 確認
- 下單瞬間鎖定庫存（pending 狀態保留 15 分鐘），逾時自動釋放，防止超賣
- 串接香港本地支付網關：信用卡（Visa/Mastercard/銀聯）、PayMe for Business、AlipayHK、WeChat Pay HK、FPS 轉數快、Apple Pay / Google Pay
- 支援優惠券 / 折扣碼於結算頁套用
- 支援訂金（Deposit）與尾款分階段收款模式（Could）
- 退款 / 取消政策引擎，依物業設定的取消條款自動計算可退金額

### 7.3 資料模型新增表（B4.2）

- `payment_transactions`：支付網關交易紀錄
- `membership_tiers`：會員等級門檻與權益
- `user_membership`：個別會員當前等級與年度消費
- `coupons`：優惠券 / 折扣碼
- `trip_plan_items`：行程規劃項目正規化子表
- `notifications_log`：通知發送紀錄
- `cms_articles`：旅遊指南 CMS 文章
- `audit_logs`：管理員操作審計紀錄

### 7.4 安全與合規（B8）

- 符合香港《個人資料（私隱）條例》(PDPO)
- 信用卡資料不經自家伺服器儲存，採用支付網關 Hosted Fields / Tokenization，降低 PCI-DSS 合規範圍至 SAQ A / A-EP
- 統一 RBAC 中介層，所有管理員寫入操作寫入 `audit_logs`
- 敏感資料（電話、護照等）儲存前加密，傳輸層強制 HTTPS/TLS 1.2+
- API 輸入採用 zod 嚴格驗證，對下單、登入等端點設置速率限制

## 8. 行動建議

基於以上摘要，後續開發可優先關注以下與原始碼差距最大的領域：

1. **支付網關整合**: 目前為最大缺口，需於 MVP 階段優先補齊。
2. **庫存並發鎖定**: 需實作 `locked_inventory` 與 15 分鐘逾時釋放機制。
3. **會員分級與優惠券**: 需新增 `membership_tiers`、`user_membership`、`coupons` 資料表與對應 API。
4. **旅遊指南 CMS**: 需將現有 `GuidePage.tsx` 靜態內容改為後台可管理。
5. **通知系統**: 需整合 WhatsApp Business API 與電郵服務，實作 D-7/D-3/D-1 行前關懷序列。
6. **審計日誌**: 所有管理員寫入操作需同步記錄至 `audit_logs`。
