# 全站「島主」相關文案梳理與替換建議

## 1. 整體建議

### 1.1 為什麼「島主」過度

目前全站把投資馬爾代夫度假酒店物業的客戶稱為「島主」，並大量使用「擁有一座島嶼」「為你的島嶼命名」「你的島嶼在等你」等文案。但實際產品是「島上度假酒店的房子/物業」，客戶並非真正擁有整座島嶼。這會帶來幾個風險：

1. **誤導性**：用戶可能誤以為購買的是整個島嶼的產權，與實際物業型態不符。
2. **合規風險**：在投資/置業語境下，誇大擁有權可能引發監管或消費者保護問題。
3. **品牌信任**：當用戶到達盡職調查階段發現只是物業單位時，會產生預期落差。

### 1.2 建議採用的方向

將核心敘事從「擁有島嶼」轉向「擁有海島度假物業 / 海島之家」，保留浪漫與體驗感，但讓所有權邊界更準確。建議優先採用以下稱謂：

| 原用法 | 建議優先用法 | 備註 |
|---|---|---|
| 島主 | **海島業主** / **海島屋主** / **度假屋主** | 核心稱謂 |
| 島主計劃 | **海島業權計劃** / **海島物業計劃** / **業主計劃** | 計劃名稱 |
| 島主體驗 | **業主體驗** / **海島業主體驗** | 體驗名稱 |
| 島主對話 | **業主對話** / **海島物業諮詢** / **物業顧問對話** | 諮詢 CTA |
| 島主關係經理 | **物業關係經理** / **海島物業顧問** / **業主關係經理** | 角色 |
| 為你的島嶼命名 | **為你的海島之家命名** / **為你的海島度假屋命名** | 互動 |
| 島主故事 | **業主故事** / **海島業主故事** / **屋主故事** | 社會證明 |
| 留下來 | **成為業主** / **擁有你的海島之家** / **決定成為屋主** | 行動暗示 |

### 1.3 預計影響頁面

- `src/layouts/Layout.tsx` — 全站導航與 CTA
- `src/pages/HomePage.tsx` — 首頁 Hero、命名互動、團隊介紹、業主故事、底部 CTA
- `src/pages/InvestPage.tsx` — 島主計劃著陸頁（影響最大）
- `src/pages/PropertyDetailPage.tsx` — 物業詳情頁的「島主視角」與體驗查詢
- `src/pages/PropertiesPage.tsx` — 物業列表頁標語與標籤
- `src/pages/MemberPage.tsx` — 會員引導頁
- `src/api/client.ts` — demo 資料中的物業名稱與描述

## 3. 逐檔掃描與分類

分類標準：

- **需要替換** — 直接稱呼客戶為島主，或暗示擁有整座島嶼。
- **可保留** — 描述性用法（如海島生活、度假村島嶼、馬爾代夫島嶼管家），不會讓用戶誤以為自己是島主。
- **需討論** — 語境模糊，需進一步判斷是否涉及產品名稱或行銷隱喻。
